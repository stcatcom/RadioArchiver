# RadioArchiver - ストレージ容量要件

## 90日間録音の容量見積もり

### デフォルト設定（ステレオ 16bit 44.1kHz）

| 期間 | 容量 |
|------|------|
| 1分 | 10.09 MB |
| 1時間 | 605 MB (0.59 GB) |
| 1日 | 14.19 GB |
| **90日** | **1,277.48 GB (約1.25 TB)** |

### 推奨ディスク容量

- **最低容量**: 1,278 GB
- **推奨容量**: 1,533 GB（20%余裕）
- **安全容量**: 1,916 GB（50%余裕）

**実用的には 2TB 以上のHDD推奨**

## 推奨ハードウェア

### 放送局（地上波・FM）

**用途**: 放送法対応（90日保存義務）

- **ディスク**: 2TB HDD または SSD
- **RAID構成**: RAID1（ミラーリング）推奨
- **実効容量**: 2TB × 2台 = 2TB（冗長化）
- **バックアップ**: 外付けHDDまたはNAS

### ネットラジオ（短期保存）

**用途**: 7日〜30日保存

- **ディスク**: 500GB SSD または 1TB HDD
- **容量**: 
  - 7日間: 約100GB
  - 30日間: 約426GB

## ストレージ最適化のヒント

### 1. 保持期間の見直し

| 保持期間 | 必要容量（44.1kHz ステレオ 16bit） |
|---------|----------------------------------|
| 7日 | 99 GB |
| 30日 | 426 GB |
| 60日 | 851 GB |
| 90日 | 1,277 GB |
| 180日 | 2,555 GB |

### 2. モノラル録音を検討

音楽放送でなければ、モノラル録音で容量を半減できます。

- ステレオ 90日: 1,277 GB
- モノラル 90日: 639 GB
- **節約**: 約50%**

### 3. サンプルレートの調整

音声放送であれば、44.1kHz → 32kHz も検討可能。

- 44.1kHz 90日: 1,277 GB
- 32kHz 90日: 927 GB
- **節約**: 約27%**

### 4. 自動削除機能の活用

RadioArchiverは古いファイルを自動削除します。

- 設定タブで保持期間を調整
- 同録ファイル: デフォルト90日
- 結合ファイル: デフォルト2時間

## よくある質問

### Q1: 2TBで足りますか？

**A**: 標準設定（ステレオ 16bit 44.1kHz）なら問題なし。
- 90日間: 1,277 GB
- 余裕: 約700GB

### Q2: SSDとHDDどちらが良い？

**A**: 用途による。
- **HDD**: コスパ重視、大容量、放送局向け
- **SSD**: 速度重視、静音、ネットラジオ向け

### Q3: RAID構成は必要？

**A**: 放送局なら推奨。
- RAID1（ミラーリング）で冗長化
- 1台故障してもデータ消失なし

### Q4: クラウドストレージは使える？

**A**: 非推奨。
- 1.3TBのアップロードは現実的でない
- コストが高額（月額数千円〜）
- ローカルストレージ推奨

### Q5: 容量不足になったら？

**A**: 以下の対策を検討。
1. 保持期間を短縮（90日→60日）
2. モノラル録音に変更
3. サンプルレートを下げる（44.1kHz→32kHz）
4. ディスクを追加購入
